# 🤖 VPN Bot v2.0 - Улучшения

## ✨ Новые функции

### 1. **Персональные защищенные ссылки**
- Каждый пользователь получает уникальный токен доступа
- Ссылка: `http://ваш-сервер:8080/sub/{user_id}/{secure_token}.txt`
- Доступ возможен **только** при активной подписке

### 2. **СБП платежи (Система Быстрых Платежей)**
- Интеграция с YooKassa/VGD
- Быстрые платежи через мобильный банк
- Подробное отслеживание платежей

### 3. **Встроенные кнопки интерфейса**
- Быстрые кнопки для управления подпиской
- Inline buttons для удобства
- Лучше структурированные сообщения

### 4. **Улучшенная база данных**
- Таблица `access_tokens` для безопасных ссылок
- Расширенная таблица `payments` с поддержкой YooKassa
- Счетчик использования токенов

### 5. **Лучший UX**
- Эмодзи в сообщениях
- Красивый интерфейс с кнопками
- Информативные ответы

## 🚀 Настройка СБП платежей

### Шаг 1: Получить учетные данные YooKassa
1. Перейти на https://yookassa.ru
2. Зарегистрировать организацию
3. Подключить СБП
4. Получить:
   - **Shop ID** (номер магазина)
   - **API Key** (секретный ключ)

### Шаг 2: Добавить в `.env`
```bash
PAYMENT_ENABLED=true
YOOKASSA_SHOP_ID=123456
YOOKASSA_API_KEY=test_AXfp21hBcFr6qznZ3z1w3ZqN23456789
```

### Шаг 3: Перезапустить бота
```bash
pkill -f "python3.*vpn_telegram_bot"
python3 tools/vpn_telegram_bot_v2.py &
```

## 💳 Обработка платежей

### Без SBP (тестовый режим)
- Кнопка "✅ Оплачено (свободный тест)"
- Подписка активируется сразу

### С SBP (реальные платежи)
1. Пользователь нажимает "💳 СБП"
2. Бот создает платеж в YooKassa
3. Пользователь переводит деньги в мобильном банке
4. Платеж подтверждается
5. Подписка активируется автоматически

## 🔐 Безопасность персональных ссылок

**До v2.0:**
```
http://localhost:8080/sub/123456.txt
❌ Любой может угадать user_id
```

**После v2.0:**
```
http://localhost:8080/sub/123456/rA7kH2pQx8bZ9mN1Y3tF5w.txt
✅ Невозможно угадать токен
✅ Доступ только при активной подписке
```

## 📊 Структура данных

### Новая таблица `access_tokens`
```sql
token TEXT PRIMARY KEY          -- Уникальный безопасный токен
user_id INTEGER                 -- ID пользователя
created_at INTEGER              -- Время создания
expires_at INTEGER              -- Время истечения (365 дней)
used_count INTEGER              -- Сколько раз использован
```

### Расширенная таблица `payments`
```sql
payment_id TEXT UNIQUE          -- Уникальный ID платежа
yookassa_id TEXT                -- ID платежа в YooKassa
```

## 📱 Callback обработка

Бот поддерживает Inline кнопки:
- `sub_7`, `sub_30`, `sub_90`, `sub_365` — подписка
- `get_link` — получить ссылку
- `status` — статус
- `pay_sbp_{дни}` — оплата через СБП
- `pay_free_{дни}` — тестовая подписка

---

## ⚠️ Важные замечания

1. **SBP платежи** требуют реального Shop ID и API Key
2. **Тестовый режим** работает без платежной системы
3. **Персональные ссылки** автоматически генерируются
4. **Администраторы** получают 365 дней бесплатно на любую команду
5. **Поддржка** обновления подписки (автоматическое расширение)
