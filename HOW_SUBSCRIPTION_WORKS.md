# 🖇️ Как VPN ссылка работает в клиентах

## 📡 Алгоритм работы

```
Пользователь получает ссылку:
  ↓
http://127.0.0.1:8080/sub/{user_id}/{token}.txt
  ↓
Вставляет в VPN клиент
  ↓
Клиент отправляет HTTP GET запрос
  ↓
Сервер проверяет:
  ✓ user_id существует?
  ✓ token валиден?
  ✓ подписка активна?
  ↓
Если ОК → возвращает конфиги (VLESS/Reality/и т.д.)
Если нет → возвращает сообщение об ошибке
  ↓
Клиент парсит конфиги и показывает список серверов
  ↓
Пользователь выбирает сервер и подключается
```

---

## 📦 Что возвращает сервер

Ссылка возвращает обычный текстовый файл:

```
# 🔐 VPN subscription for user 123456
# ⏰ Expires: 2026-04-18 12:00:00
# 📅 Remaining: 7 days

vless://90c9c25c-...@147.45.227.198:443?encryption=none&...#Netherlands
vless://8cf1c214-...@fra2.example.com:443?encryption=none&...#Germany
vless://3dfe4106-...@185.121.233.103:8443?type=xhttp&...#Amsterdam
...
```

Каждая строка = один VLESS конфиг для сервера

## 🔄 Как клиенты используют ссылку

### Clash

```yaml
# Когда пользователь вставляет URL:
# URL: http://127.0.0.1:8080/sub/123456/token.txt

# Clash каждый час:
1. Отправляет GET запрос на URL
2. Получает текстовый файл с конфигами
3. Парсит каждую строку как proxy
4. Обновляет список доступных серверов

# В результате в Clash:
├─ Proxy 1 (Netherlands)
├─ Proxy 2 (Germany)  
├─ Proxy 3 (Amsterdam)
└─ ... и так далее
```

### Shadowrocket (iOS)

```
Пользователь в Shadowrocket:
1. Нажимает "Subscribe" (+)
2. Вставляет URL подписки
3. Shadowrocket загружает конфиги
4. Показывает список серверов в интерфейсе

Каждая подписка обновляется автоматически:
- Каждый раз при открытии приложения
- Или вручную через Refresh
```

### Clash for Android

```
1. Вкладка Profiles (Профили)
2. Нажимает + (Add Profile)
3. Вставляет URL
4. Нажимает Save/Confirm
5. Профиль загружается и список обновляется

Обновление: вручную или автоматически
```

## 🔐 Безопасность персональной ссылки

### Что защищает ссылка:

```
НЕПРАВИЛЬНО ❌
http://localhost:8080/sub/123456.txt
  ↳ Любой может угадать user_id 123456
  ↳ Может получить чужие конфиги
  ↳ Может заставить сервер работать на всех

ПРАВИЛЬНО ✅
http://localhost:8080/sub/123456/rA7kH2pQx8bZ9mN1Y3tF5w.txt
  ↳ Невозможно угадать токен (32-символьный случайный)
  ↳ Только владелец может использовать
  ↳ Каждый пользователь = свой уникальный токен
```

### Что проверяет сервер:

```python
def verify_token(token):
    # Проверка 1: Токен существует в БД?
    ✓ SELECT ...WHERE token = ?
    
    # Проверка 2: Токен не истек?
    ✓ WHERE expires_at > NOW()
    
    # Проверка 3: user_id верный?
    ✓ токен belongs to user_id
    
    # Проверка 4: Подписка активна?
    ✓ SELECT subscriptions WHERE status='active'
    
    # Если все ОК → возвращаем конфиги
    # Если нет → возвращаем ошибку 403 Forbidden
```

## 🚀 Практический пример

### Сценарий: Пользователь активирует подписку

```
1. Пользователь в Telegram боте:
   /subscribe 7
   
2. Бот создает в БД:
   - subscriptions: expires_at = now + 7 дней
   - payments: amount = 7 RUB, status = completed
   - access_tokens: token = "rA7kH2...", user_id = 123456
   
3. Бот отправляет пользователю:
   ✅ Подписка активирована на 7 дней
   🔗 http://localhost:8080/sub/123456/rA7kH2.txt
   
4. Пользователь копирует ссылку и вставляет в Clash
   
5. Clash отправляет на сервер:
   GET /sub/123456/rA7kH2.txt HTTP/1.1
   
6. Сервер проверяет:
   ✓ Token rA7kH2 существует → user_id = 123456
   ✓ Токен не истек (expires_at = 2026-04-25)
   ✓ Подписка активна (expires_at = 2026-04-18)
   ✓ HTTP 200 OK
   
7. Сервер возвращает:
   ```
   # VPN configs
   vless://... (Netherlands)
   vless://... (Germany)
   vless://... (Amsterdam)
   ```
   
8. Clash парсит и показывает список:
   [Proxy] Netherlands ← пользователь выбирает
   [Proxy] Germany
   [Proxy] Amsterdam
   
9. Пользователь нажимает Netherlands и подключается!
```

## 🔁 Обновление конфигов

### Как клиент обновляет конфиги?

```
Clash:
- Автоматически: каждый час
- Или вручную: Profiles → Refresh

Shadowrocket:
- Автоматически: при открытии приложения
- Или вручную: Subscriptions → Pull to refresh

Hiddify:
- Автоматически: каждый час по умолчанию
- Или вручную: Profile → Refresh
```

### Что обновляется?

```
Ссылка всегда возвращает актуальные конфиги:
- Если администратор добавил новый сервер
- Если сервер удален/отключен
- Если обновлены параметры сервера

Все пользователи:
1. Не должны переводить ссылку
2. Один раз добавили подписку в клиент
3. Клиент сам все обновляет!
```

## ⚙️ HTTP заголовки

Когда клиент запрашивает ссылку, сервер отправляет:

```
HTTP/1.1 200 OK
Content-Type: text/plain; charset=utf-8
Content-Length: 5234
Cache-Control: no-cache

[здесь конфиги]
```

Это позволяет клиентам правильно парсить файл.

## 🚨 Ошибки и их решение

### Ошибка: "Invalid subscription URL"
```
Возможные причины:
1. URL скопирована неправильно (пробелы, переносы)
2. Пользователь офлайн / нет интернета
3. Сервер недоступен

Решение:
- Пересчитайте ссылку из бота (/link)
- Убедитесь подписка активна (/status)
- Проверьте интернет
```

### Ошибка: "Subscription expired"
```
Причина:
- Прошло 7+ дней и подписка истекла

Решение:
- Нужна новая подписка (/subscribe)
```

### Ошибка: "403 Forbidden"
```
Причина:
1. Токен неправильный
2. Подписка неактивна
3. Токен истек

Решение:
- Запросите новую ссылку у бота (/link)
```

## 🔧 Для администратора

### Как тестировать ссылку пользователя:

```bash
# Получить ссылку пользователя
user_id=123456
token="rA7kH2pQx8bZ9mN1Y3tF5w"

# Тест 1: Должно работать
curl "http://localhost:8080/sub/$user_id/$token.txt" | head -5

# Тест 2: Неправильный токен - должно быть 403
curl "http://localhost:8080/sub/$user_id/wrongtoken.txt"

# Тест 3: Проверить размер файла
curl -I "http://localhost:8080/sub/$user_id/$token.txt"

# Тест 4: Сохранить конфиги локально
curl "http://localhost:8080/sub/$user_id/$token.txt" > my_configs.txt
```

### Если ссылка не работает в программе:

1. Проверьте что конфиги правильные (VLESS формат)
2. Тестите через curl
3. Проверьте логи: `tail -f /tmp/bot_v2.log`
