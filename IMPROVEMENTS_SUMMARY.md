# 📊 Итоги улучшений алгоритма выбора серверов

## Дата реализации
10 апреля 2026 г.

## 🎯 Основные улучшения

### 1. **Новая система ранжирования (Weighted Ranking)**
- Вместо простой сортировки по ping + probe_ok
- Теперь использует **взвешенный скор** с несколькими факторами:
  - ✅ Статус TLS проверки (вес: 1000)
  - ⚡ Низкий пинг (вес: 10)
  - 📈 Стабильность (вес: 5)
  - ✔️ Процент успехов (штраф: 0-50)

### 2. **Метрики стабильности соединения**
Для каждого тестируемого сервера теперь вычисляются:
- **stdev** — стандартное отклонение пинга (мс)
- **cv%** — коэффициент вариации в процентах
- **spread** — максимальный разброс (max - min) пингов
- **success_rate** — доля успешных подключений

### 3. **Расширенные опции фильтрации**
```bash
--max-cv <число>           # Фильтр по максимальной вариативности (%)
--max-ping-spread <мс>     # Фильтр по максимальному разбросу пингов
```

### 4. **Улучшенные отчеты**
- **ping_top.tsv** — теперь содержит все метрики стабильности
- **report.json** — включает раздел "stability" с объяснениями метрик

### 5. **Исправления в скрипте обновления**
- Заменена ненадежная операция `git pull --rebase` на более стабильный `git pull --no-rebase`
- Добавлены флаги `|| true` для graceful обработки ошибок

## 🎓 Как можно использовать улучшения

### Для максимальной надежности
```bash
python3 tools/vpn_picker.py --max-cv 10 --tries 5
```
Выберет только очень стабильные серверы (вариативность < 10%)

### Для баланса скорости и стабильности
```bash
python3 tools/vpn_picker.py --max-cv 20 --tries 3
```
Выберет быстрые И достаточно стабильные серверы

### Для мобильных пользователей
```bash
python3 tools/vpn_picker.py --max-ping-spread 1.0 --tries 5
```
Выберет серверы с низким разбросом пингов (хорошо для нестабильных сетей)

## 🔧 Технические детали

### Измененные функции в vpn_picker.py

1. **`calculate_rank_score()`** (новая)
   - Вычисляет взвешенный скор для ранжирования
   - Нормализует значения в диапазон 0-100
   - Применяет разные веса к разным факторам

2. **`measure_candidate()`** (модифицирована)
   - Теперь вычисляет метрики стабильности
   - Добавляет поле `_stability_score` к каждому результату

3. **Аргументы CLI** (расширены)
   - Добавлены `--max-cv` и `--max-ping-spread`
   - Фильтрация применяется перед финальной выборкой TOP-N
