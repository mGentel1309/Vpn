# 🚀 VPN Server Auto-Picker

Автоматическое получение, проверка и отбор самых быстрых VPN серверов из репозитория [zieng2/wl](https://github.com/zieng2/wl).

## 📋 Возможности

✅ **Автоматическое обновление** конфигов каждые 6 часов (GitHub Actions)  
✅ **Тестирование скорости** через ICMP пинг  
✅ **Отбор топ-10** самых быстрых серверов  
✅ **Сохранение результатов** в `top-10.txt`  
✅ **Git интеграция** - автоматический commit и push  

## 🔧 Структура

```
.
├── tools/
│   ├── fetch-and-pick.sh      # Основной скрипт для отбора серверов
│   └── vpn_picker.py          # Python тулз для тестирования
├── .github/workflows/
│   └── auto-pick-servers.yml  # GitHub Actions (запуск каждые 6 часов)
├── vless.txt                  # Полный список конфигов
├── vless_lite.txt             # Облегченная версия
├── vless_universal.txt        # Универсальная версия
└── top-10.txt                 # Результат: топ-10 быстрых серверов
```

## ⏱️ Время выполнения и метод валидации

| Режим | Метод пинга | Время | Сортировка | Файл | Качество |
|--------|--------|--------|---------|------|---------|
| ⚡ **Срочно** | TCP connect | 2-3 мин | O(n) | fetch-and-pick-fast.sh | Низкое |
| ✨ **Быстро** | TCP handshake | 5-8 мин | O(n) | fetch-and-pick.sh | Среднее |
| 🚀 **Продвинуто** | Реальный ICMP | 5-8 мин | O(n log n) | fetch-and-pick-advanced.sh | **Высокое** |
| 🏆 **Лучше всех** | Реальный ICMP | 8-12 мин | O(n log n) | fetch-and-pick-best.sh | **Максимум** |
| 🔬 **Строго** | TCP handshake | 10-15 мин | O(n) | fetch-and-pick-strict.sh | Высокое |

**⭐ Рекомендация:** Используй `fetch-and-pick-advanced.sh` или `fetch-and-pick-best.sh`!
- Реальный ICMP пинг как в Happ
- O(n log n) сортировка
- Надежный результат

## 🚀 Использование

### 4 режима запуска (от быстрого к лучшему):

#### 1. **Срочно** ⚡ (2-3 мин) - АВАРИЙНЫЙ
```bash
./tools/fetch-and-pick-fast.sh
```
- Минимум проверок
- Только для "дать интернет сейчас"

#### 2. **Быстро** (5-8 мин) - БЫСТРО ✨ 
```bash
./tools/fetch-and-pick.sh
```
- TLS валидация
- Оптимальный баланс скорость/качество

#### 3. **Продвинуто** 🚀 (5-8 мин) - РЕКОМЕНДУЕТСЯ
```bash
./tools/fetch-and-pick-advanced.sh
```
- **Реальный ICMP ping** (как Happ)
- **O(n log n) сортировка** по латенси
- **3 попытки пинга** для каждого сервера
- TCP fallback
- 70%+ success rate фильтр
- **ЛУЧШИЙ РЕЗУЛЬТАТ**

#### 4. **Лучше всех** 🏆 (8-12 мин) - PRODUCTION
```bash
./tools/fetch-and-pick-best.sh
```
- Комбинирует все подходы
- Максимальная надежность
- Для критичных задач

#### 5. **Строго** 🔬 (10-15 мин) - МАКСИМУМ ПРОВЕРОК
```bash
./tools/fetch-and-pick-strict.sh
```
- 2 попытки пинга
- 200 серверов для проверки
- С повторами

### GitHub Actions (автоматически)

- ⏰ Запускается **каждые 6 часов** 
- Использует **"продвинуто" режим** (реальный пинг)
- Автоматически коммитит результаты

Запустить вручную: **GitHub → Actions → Auto-Update & Pick Best Servers → Run workflow**

## ⚙️ Параметры

| Переменная | По умолчанию | Описание |
|-----------|-------------|---------|
| `TRIES` | 1 | Количество попыток проверки каждого сервера |
| `TOP` | 10 | Количество серверов в результате |
| `SUBSCRIPTION` | `vless_universal.txt` | Исходный файл конфигов |
| `LIMIT` | 150 | Макс. серверов для проверки |
| `TIMEOUT` | 2.0 | Таймаут соединения (сек) |
| `CONCURRENCY` | 100 | Количество одновременных проверок |
| `RETRY` | 0 | Повтор при недостаточном кол-ве (1=да, 0=нет) |

### Режимы по скоростям:

| Режим | Скорость | Пинг | Сортировка | LIMIT | Файл |
|-------|---------|------|---------|--------|------|
| ⚡ **Срочно** | 2-3 мин | TCP | O(n) | 50 | fetch-and-pick-fast.sh |
| ✨ **Быстро** | 5-8 мин | TCP | O(n) | 150 | fetch-and-pick.sh |
| 🚀 **Продвинуто** | 5-8 мин | **ICMP** | **O(n log n)** | 150 | fetch-and-pick-advanced.sh ⭐ |
| 🏆 **Лучше** | 8-12 мин | **ICMP** | **O(n log n)** | 200 | fetch-and-pick-best.sh ⭐ |
| 🔬 **Строго** | 10-15 мин | TCP | O(n) | 200 | fetch-and-pick-strict.sh |

### Свои параметры:

```bash
# Проверяет 300 серверов, ищет топ-20
LIMIT=300 TOP=20 ./tools/fetch-and-pick.sh

# Очень быстро (50 серверов, 50 параллельных)
RETRY=0 LIMIT=50 CONCURRENCY=50 ./tools/fetch-and-pick.sh

# С повторами для гарантии результата
RETRY=1 LIMIT=200 TRIES=2 ./tools/fetch-and-pick.sh
```

## � Результаты

Результаты сохраняются в два файла:

- **`vpn.txt`** ← 🌟 **ОСНОВНОЙ ФАЙЛ** (используй этот)
- **`top-10.txt`** ← Резервная копия (совпадает с vpn.txt)

Оба файла содержат топ-N самых быстрых + проверенных серверов:


- Проверку TCP соединения
- Проверку TLS сертификата (для reality/TLS конфигов)
- Измерение задержки (ping)
- Фильтр по стабильности

## 🎯 Как работает "Продвинуто" и "Лучше всех" режимы

### Отличие от других:
1. **Реальный ICMP пинг** вместо TCP handshake
   - 3 попытки для каждого сервера
   - Как проверяет Happ приложение
   - Фильтр: >70% успешных попыток

2. **O(n log n) сортировка** (Python Timsort)
   - Быстрая и надежная
   - Результаты отсортированы по реальной задержке
   - Гарантирует самые быстрые в начале

3. **TCP fallback**
   - Если ICMP недоступен, использует TCP
   - Гарантирует результат даже на ограниченных сетях

### Результат:
- ✅ Только работающие серверы (проверено многократно)
- ⚡ Правильно отсортированы по скорости
- 🎯 Как в production-ready приложении Happ

## �📞 Troubleshooting

**Нет результатов?**
- Проверьте интернет соединение
- Убедитесь что `vless_universal.txt` содержит конфиги
- Запустите с `TRIES=1` для быстрого теста

**Push failed?**
- Проверьте GitHub token доступ
- Убедитесь что репо позволяет write access

**Нужна помощь?**
- Смотрите логи GitHub Actions: Settings → Actions
- Запустите скрипт локально для отладки
